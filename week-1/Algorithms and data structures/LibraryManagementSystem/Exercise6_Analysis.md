Time Complexity
Linear Search: O(n) because books are checked one by one.
Binary Search: O(log n) because it divides the search space into halves.
When to Use Each
Linear Search: Best for small collections or unsorted data.
Binary Search: Best for large collections when the books are stored in sorted order.
Suitable for a Library System

For a large library, Binary Search is more suitable because it finds books much faster than Linear Search. However, the book records must be sorted before using it.
