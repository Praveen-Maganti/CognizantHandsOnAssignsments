Time Complexity
Add: O(n) because we may need to move to the end of the list.
Search: O(n) because tasks are checked one by one.
Traverse: O(n) because every task is visited.
Delete: O(n) because the task must be found before deleting it.
Advantages of Linked Lists
Dynamic Size: Tasks can be added or removed easily without a fixed size limit.
Easy Insertion and Deletion: Adding or removing tasks is simpler than in arrays because elements do not need to be shifted.
Suitable for a Task Management System

Linked lists are useful when tasks are frequently added or removed, as they provide more flexibility than arrays.
